# 多线程渲染框架
1. 主线程：计算逻辑，通过Renderer::Submit()提交渲染命令缓存
2. 每次收集完一帧的命令后，Kick()渲染线程开始把渲染命令全部执行，同时主线程开始收集下一帧的渲染命令（单线程模式，等待渲染命令执行完后才开始收集下一帧命令）
3. 所以：上一帧的渲染命令执行和本帧的渲染命令收集是并发执行的
4. 整个系统的设计的并发有两层：（CPU 主线程 ↔ 渲染线程并行）  （渲染线程 ↔ GPU并行（多帧飞行技术））
